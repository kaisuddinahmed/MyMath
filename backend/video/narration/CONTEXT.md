# Context for finetuning video scenes and narrations — MyMath

This is a children's math video app (NCTB Class 1, age ~6). Videos are generated by a deterministic pipeline — no LLM involved in scene or narration generation.

## How it works end to end:

1. `backend/video/narration/nctb/class_1.py` — builds narration beats per topic. Each beat has: `text` (spoken audio), `action` (which visual scene plays), `equation` (shown on screen), `item_type` (SVG icon e.g. `FLOWER_SVG`), `extra` (additional scene params)
2. Each beat → one Remotion scene. Three consecutive `SHOW_SMALL_ADDITION` beats group into one `SmallAdditionScene` with three animation phases
3. TTS audio length drives scene duration — longer narration = longer scene
4. `remotion/src/compositions/MathVideo.tsx` — master compositor, routes each action to its scene component
5. `remotion/src/components/Scenes/` — one `.tsx` file per visual scene type
6. `remotion/src/components/Engines/ObjectArray.tsx` — the core animation engine for item groups (controls pop-in, gap, merge, + sign)

## Action → Scene file mapping (most relevant for Class 1):

- `SHOW_SMALL_ADDITION` → `SmallAdditionScene.tsx` → uses `ObjectArray.tsx`
- `CHOREOGRAPH_SUBTRACTION` → `ChoreographyScene.tsx`
- `SHOW_PART_WHOLE_SUBTRACTION` → `PartWholeScene.tsx`
- `SHOW_MEDIUM_SUBTRACTION` → `MediumSubtractionScene.tsx`
- `ADD_ITEMS` / `REMOVE_ITEMS` / `HIGHLIGHT` → `Scenes.tsx` → `CounterScene`
- `DRAW_SHAPE` → `GeometryScene.tsx`

## Quick reference:

- **To change what the child hears:** edit `text=` in `nctb/class_1.py`
- **To change what appears on screen:** edit the `.tsx` scene file for that action
- **To change animation timing within a scene:** edit the `timings` / `spring` / `interpolate` logic in the relevant `.tsx`
- **To fix audio/video sync:** `MathVideo.tsx` — `0.5 * fps` (audio start delay), `1.5 * fps` (padding after audio ends)
- **Reference doc:** `PROJECT_MAP.md` in the MyMath root — full file-to-function mapping
